Lemma 26 Increasing-pair count
✓
#

Lean declarations

  • LeaFrontier.OrderingRecovery.card_lt_pairs

\(\# \{ (i,j)\in (\mathrm{Fin}\, m)^2 : i{\lt}j\} = \binom m2\).

Proof ▶

Of the \(m^2\) ordered pairs, \(m\) lie on the diagonal and the remaining \(m^2 - m\) split evenly between \(i{\lt}j\) and \(i{\gt}j\), giving \(\binom m2 = m(m-1)/2\).

Definition 27 Induced vertex order

For injective \(g : \mathrm{Fin}\, n \to \alpha \) into a linear order, \(\mathrm{rankPerm}\, g\) ranks \(\mathrm{Fin}\, n\) by \(g\): \(\mathrm{rankPerm}\, g\, u {\lt} \mathrm{rankPerm}\, g\, v \iff g(u) {\lt} g(v)\) (in Lean, \((\mathrm{Tuple.sort}\, g)^{-1}\)). The induced vertex order is \(\pi _{\hat\pi } = \mathrm{rankPerm}(u \mapsto \hat\pi (d_u))\).

  • LeaFrontier.OrderingRecovery.objective_identity

With \(C_0 = m(\tfrac 12 - \tfrac 1{R^2}) + (\binom L2 - m)\tfrac 12\) and \(\pi _{\hat\pi }\) the induced vertex order,

\[ \mathrm{cost}(\hat\pi ) = C_0 + \frac{2}{R^2}\, \operatorname {fas}(T, \pi _{\hat\pi }). \]
Proof ▶

Apply the pairwise decomposition. A designated pair \(\{ d_u,d_v\} \) contributes, via Lemma 21, constant term \(\tfrac 12 - \tfrac 1{R^2}\) plus decision weight \(\tfrac 2{R^2}\) times the indicator that \(\hat\pi \) disagrees with the majority order; by construction this disagreement happens exactly when the arc is backward under \(\pi _{\hat\pi }\). Every filler-involved pair contributes exactly \(\tfrac 12\) (Lemma 25), with zero decision weight. Summing the \(m\) designated constant terms and the \(\binom L2 - m\) filler terms gives \(C_0\); the decision weights sum to \(\tfrac 2{R^2}\) times the number of backward arcs, i.e. \(\operatorname {fas}(T,\pi _{\hat\pi })\). Reindexing designated pairs to \(\mathrm{Fin}\, n\) pairs and counting via Lemma 26 closes it.

Lemma 29 Reduction correctness
✓
#

Uses

  • Lemma 28
  • Definition 3
  • Definition 14
  • Lemma 28

Lean declarations

  • LeaFrontier.OrderingRecovery.reduction_correct

With threshold \(K = C_0 + 2k/R^2\),

\[ \mathrm{FAST}(T,k) \iff \mathrm{ORP}(f(T,k), K). \]
Proof ▶

Forward: given \(\pi '\) with \(\operatorname {fas}(T,\pi ') \le k\), extend it to any full event order \(\hat\pi \) whose restriction to the designated events is \(\pi '\) (via Equiv.Perm.extendDomain, so the induced order is exactly \(\pi '\)). By the objective identity (Lemma 28), \(\mathrm{cost}(\hat\pi ) = C_0 + \tfrac 2{R^2}\operatorname {fas}(T,\pi ') \le C_0 + \tfrac {2k}{R^2} = K\). Backward: from any \(\hat\pi \) with \(\mathrm{cost}(\hat\pi ) \le K\), the identity gives \(C_0 + \tfrac 2{R^2}\operatorname {fas}(T,\pi _{\hat\pi }) \le C_0 + \tfrac {2k}{R^2}\); clearing the positive factor \(\tfrac 2{R^2} {\gt} 0\) (multiply by \(R^2\), then linarith) yields \(\operatorname {fas}(T,\pi _{\hat\pi }) \le k\). The \(n = 0\) empty tournament is handled separately (both sides hold trivially).

Lemma 30 Polynomial size bounds
✓
#

Uses

  • Definition 14
  • Lemma 13

Lean declarations

  • LeaFrontier.OrderingRecovery.poly_size_bounds

\(m \le n^2\), \(R = 2m + 8n \le 2n^2 + 8n\), and \(L = 2nR \le 4n^3 + 16n^2\); in particular \(L \le 16(n+1)^3\), so the reduction’s output is polynomial-sized.

Proof ▶

Direct arithmetic from \(m = \binom n2 \le n^2\) and \(R = 2m+8n\), \(L = 2nR\). The explicit bound \(L = 2nR \le 16(n+1)^3\) is what the headline theorem consumes as its polynomial-size certificate \(\langle 16, 3\rangle \).
